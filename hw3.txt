Вопрос 3 

Бит setgid устанавливает владельца файла или каталога. Применяется для организации совместной работы пользователей, 
например в общей папке.
Примерный сценарий:
Создаем папку /WORK
sudo mkdir /WORK
Создаем общую группу:
sudo addgroup work
Добавляем пользователей в данную группу:
sudo addgroup ivan work
sudo addgroup vasya work
Проверяем попалили пользователи в группу work:
less /etc/group | grep work
Вывод будет примерно такой:
work:x:1007:ivan,vasya
Меняем группу-владельца у каталога /WORK на work
sudo chown root:work /WORK/
ls -l / | grep WORK
Вывод будет похожий на это:
drwxr-xr-x   2 root work   4096 2020-05-11 13:19 WORK
Устанавливаем права на каталог
sudo chmod g+ws,o= /WORK/
Проверяем измененные права
 ls -l / | grep WORK
Вывод примерно такой:
drwxrws—   2 root цщкл   4096 2020-05-1 13:29 WORK
Теперь у создаваемых файлов будет владелец группа work


Вопрос 4
Клонируем репозиторий ноды:
sudo git clone https://github.com/nodejs/node.git /opt/nodejs_git
Переходим в каталог локального репозитория ноды:
cd /opt/nodejs_git
Каталог /opt предназначен для размещения готовых приложений. Не для сборки. 
Устанавливаем программы, необходимые для сборки и конфигурации ноды
sudo apt install python g++ make python3-distutils
Выполняем скрипт конфигурации:
sudo ./configure
Выполняем сборку:
sudo make -j4

Последнее - запускаем установку:
sudo make install

Результатом выполнения команды будет размещение исполняемого файла в каталоге /usr/sbin/node
Если не выполнять последнюю команду, то скомпилированный файл можно запускать из каталога
/opt/nodejs_git/out/Release/node

Или добавить в ./bashrc
export PATH="/opt/nodejs_git/out/Release:$PATH"
